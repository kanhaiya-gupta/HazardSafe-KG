Example Knowledge Graph

Real-World Hazardous Substance Management

This example shows how the HazardSafe-KG system represents real hazardous substances and their relationships in a knowledge graph. The diagram demonstrates:

📊 Node Types (6 categories):

  • Chemical Substances: H2SO4, NaOH, HCl, Toluene, Acetone with detailed properties
  • Hazard Classes: Corrosive and Flammable hazards with GHS symbols
  • Storage Locations: Laboratory and warehouse storage with specifications
  • PPE Requirements: Gloves and goggles with material specifications
  • Safety Procedures: Spill response and waste disposal protocols
  • Regulations: OSHA and EPA compliance requirements

🔗 Relationship Types (7 categories):

  • HAS_HAZARD_CLASS: Chemicals linked to their hazard classifications
  • STORED_IN: Chemicals linked to their storage locations with quantities
  • REQUIRES_PPE: Hazard classes linked to required protective equipment
  • HAS_PROCEDURE: Chemicals linked to safety procedures
  • REGULATES: Regulations linked to applicable chemicals
  • INCOMPATIBLE_WITH: Chemicals that cannot be stored together
  • COMPATIBLE_WITH: Chemicals that can be safely stored together

🎯 Key Benefits Demonstrated:

  • Safety Compliance: Automatic PPE requirements based on hazard classes
  • Storage Management: Track chemical locations, quantities, and containers
  • Emergency Response: Quick access to spill response procedures
  • Regulatory Compliance: OSHA and EPA requirements automatically linked
  • Chemical Compatibility: Prevent incompatible chemical storage
  • Waste Management: Proper disposal procedures for each chemical

🔍 Example Queries This Enables:

  • "What PPE is required for handling sulfuric acid?"
  • "Where is toluene stored and in what quantity?"
  • "What procedures apply to corrosive chemicals?"
  • "Which chemicals are incompatible with sodium hydroxide?"
  • "What regulations apply to flammable substances?"

This knowledge graph serves as the central intelligence hub, connecting all safety information and enabling intelligent querying and decision-making for hazardous substance management.

Component Legend

Input Sources: PDFs, Ontology files, CSV data
RAG Processing: Document extraction, vector storage
Ontology: Schema definition, validation rules
Knowledge Graph: Neo4j database, relationships
AI Components: NLP, embeddings, reasoning
Output: User responses, insights

Technology Stack

Backend Technologies

  • FastAPI: High-performance web framework for building APIs
  • Neo4j: Graph database for knowledge graph storage
  • RDFLib: Python library for working with RDF and ontology files
  • Sentence Transformers: AI-powered text embeddings for semantic search
  • PyPDF2/PDFMiner: PDF text extraction and processing

AI & Machine Learning

  • Natural Language Processing: Named Entity Recognition, Part-of-Speech tagging
  • Semantic Embeddings: Vector representations for similarity search
  • Document Classification: AI-powered content categorization
  • Query Understanding: Intent classification and entity extraction
  • Answer Synthesis: Intelligent response generation from multiple sources

Frontend & Visualization

  • Flask: Web framework for serving the user interface
  • D3.js/Cytoscape.js: Interactive knowledge graph visualization
  • Bootstrap: Responsive UI components and styling
  • Chart.js: Data visualization and analytics
